DEPENDENCY DIAGRAM:
════════════════════════════════════════════════════════════════════════════════

When game starts:

1. MainServer_Phase4_Extended loads
   ├─ Requires GhostAI ✅
   └─ Requires Constants, Config, GhostData, ZoneData

2. ZoneManager loads
   ├─ Requires PhaseManager ✅
   └─ Requires ZoneData

3. ZoneManager:initialize() called
   ├─ Creates PhaseManager instance
   ├─ PhaseManager:initialize() creates private phases for players
   └─ Stores phaseManager in self.phaseManager

4. startGhostSpawnLoop() called
   ├─ Spawns ghosts in shared zones
   ├─ Spawns ghosts in player private phases (using phaseManager reference)
   └─ GhostAI:initializeGhost() called for each ghost

5. Zone detection runs
   ├─ Checks if player in private phase (via phaseManager)
   └─ Falls back to shared zone detection

VISUAL: HOW PHASEMANAGER INTEGRATES:
════════════════════════════════════════════════════════════════════════════════

BEFORE (Old System):

  Player Joins
      ↓
  Spawned in shared "Hub" zone
      ↓
  All players see same ghosts
      ↓
  Can't have private home zones

AFTER (New System with PhaseManager):

  Player Joins
      ↓
  PhaseManager creates Phase_1_PlayerName folder
      ↓
  Clones Hub zone into Phase_1_PlayerName
      ↓
  Teleports player to their private phase
      ↓
  Ghost spawner spawns in:
      • Shared zones (Whisper Woods, Foggy Fields, etc.)
      • Player's private Hub phase
      ↓
  Zone detector checks:
      • Is player in private phase? (via phaseManager:isZoneInPhase)
      • If yes → "Starting Area"
      • If no → Check shared zones
      ↓
  Player can catch ghosts in private phase
      ↓
  Data saved to DataStore
      ↓
  Player rejoin:
      • New phase created
      • Previous data loaded
      • Ready to catch more!

KEY POINTS:
════════════════════════════════════════════════════════════════════════════════

✅ PhaseManager creates folders in WORKSPACE (not ReplicatedStorage)
   └─ Workspace/Phase_1_PlayerName/Hub (cloned)

✅ Each Phase folder is TEMPORARY
   └─ Deleted when player leaves (no data loss, data in DataStore)

✅ GhostAI doesn't need separate folder
   └─ Single module, simple behavior definitions

✅ Both PhaseManager and GhostAI require from ServerScriptService
   └─ Must be in ServerScriptService, exact names matter

✅ Spawn loop waits for initialization
   └─ startGhostSpawnLoop() only called AFTER ZoneManager:initialize()
   └─ This ensures phaseManager exists before spawning

════════════════════════════════════════════════════════════════════════════════

COMMON MISTAKES:
════════════════════════════════════════════════════════════════════════════════

❌ Creating GhostAI as a Script instead of ModuleScript
   → Use the grey ModuleScript icon, not blue Script icon

❌ Putting GhostAI/PhaseManager in a folder
   → They must be directly in ServerScriptService

❌ Misspelling module names (e.g., "GhostAI.lua" instead of "GhostAI")
   → Studio will look for exact name "GhostAI"

❌ Not replacing ENTIRE MainServer/ZoneManager files
   → Copy/paste the complete file, not just parts

❌ Forgetting to create one of the modules
   → "Module required but not found" error will appear

✅ CORRECT:
   ServerScriptService/
   ├─ GhostAI (ModuleScript)
   ├─ PhaseManager (ModuleScript)
   ├─ MainServer_Phase4_Extended (Script)
   └─ ZoneManager (ModuleScript)
